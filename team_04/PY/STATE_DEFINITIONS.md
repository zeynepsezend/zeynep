# AgentState - Complete Reference Documentation

## Overview

The `AgentState` TypedDict defines all data that flows through the design generation workflow. This document provides detailed specifications for each state variable.

---

## State Variables by Phase

### PHASE 1: INPUT & CONTEXT SETUP

#### `user_prompt: str`
- **Purpose**: Store the user's design instruction
- **Source**: Command-line argument passed to `run_agent()`
- **Example**: `"Optimize the kitchen layout for better workflow"`
- **Used By**: Input Setup node, Suggestion Layer
- **Modified By**: Input Setup node

#### `user_input: str`
- **Purpose**: Alternative storage for user input
- **Source**: Copied from `user_prompt` in Input Setup
- **Usage**: Backup reference for user intention
- **Modified By**: Input Setup node

#### `layout_json_string: str`
- **Purpose**: JSON string representation of current layout
- **Format**: Valid JSON with rooms, dimensions, constraints
- **Source**: Loaded from layout files during bootstrap
- **Example**:
  ```json
  {
    "rooms": [
      {"name": "kitchen", "dimensions": [4, 5], "area": 20},
      {"name": "living", "dimensions": [6, 8], "area": 48}
    ],
    "constraints": []
  }
  ```
- **Used By**: All nodes for context
- **Read-Only**: Yes (avoid modifications)

#### `layout_data: dict[str, Any]`
- **Purpose**: Parsed layout data structure
- **Contents**: Same data as `layout_json_string` but as Python dict
- **Structure**:
  ```python
  {
    "rooms": list,           # Room definitions
    "doors": list,           # Door locations
    "windows": list,         # Window positions
    "constraints": list,     # Design constraints
    "metadata": dict         # Layout metadata
  }
  ```
- **Used By**: All analysis nodes
- **Mutable**: No

#### `cached_scene_state: dict[str, Any] | None`
- **Purpose**: Previously saved design state for continuity
- **Value When Null**: First iteration or no cache available
- **Contents When Set**:
  ```python
  {
    "shapes": list,          # Previous shape proposals
    "metrics": dict,         # Previous performance metrics
    "constraints_satisfied": bool,  # Prior constraint status
    "timestamp": str,        # When state was saved
    "iteration": int         # Iteration number
  }
  ```
- **Used By**: Visualization, Optimization suggestion
- **Set By**: Input Setup (loads if available)

#### `messages: list[dict[str, Any]]`
- **Purpose**: Conversation history in OpenAI format
- **Structure**: List of {"role": str, "content": str} dicts
- **Roles**: "user", "assistant", "system"
- **Example**:
  ```python
  [
    {"role": "user", "content": "Optimize kitchen layout"},
    {"role": "assistant", "content": "I'll analyze..."}
  ]
  ```
- **Used By**: LLM nodes for context
- **Maintained By**: Reason node (appends responses)

### PHASE 2: GENERATION ANALYSIS LOOP

#### `suggestions: list[dict[str, Any]]`
- **Purpose**: Design suggestions generated by LLM
- **Structure**: Each suggestion object contains:
  ```python
  {
    "id": str,                      # Unique identifier
    "type": str,                    # "shape_creation", "optimization", etc.
    "description": str,             # Human-readable description
    "confidence": float,            # 0.0 to 1.0 confidence score
    "justification": str,           # Why this is suggested
    "parameters": dict,             # Suggestion parameters
    "alternatives": list            # Alternative approaches
  }
  ```
- **Initial**: Empty list `[]`
- **Populated By**: Suggestion Layer node
- **Used By**: Shape Creation, Optimization nodes
- **Example**:
  ```python
  {
    "id": "sug_001",
    "type": "shape_creation",
    "description": "Create 2m² prep island in center",
    "confidence": 0.87,
    "justification": "Improves workflow efficiency",
    "parameters": {"width": 2, "depth": 1, "material": "granite"}
  }
  ```

#### `current_suggestion_index: int`
- **Purpose**: Track which suggestion is currently being processed
- **Initial Value**: `0`
- **Range**: `0` to `len(suggestions) - 1`
- **Usage**: When multiple suggestions need sequential processing
- **Modified By**: Suggestion Layer, Optimization nodes
- **Example**: Processing suggestion #2 of 5 suggestions

#### `proposed_shapes: list[dict[str, Any]]`
- **Purpose**: Geometric shapes generated from suggestions
- **Structure**: Each shape contains:
  ```python
  {
    "id": str,                      # Shape identifier
    "type": str,                    # "rectangle", "polygon", "curve", etc.
    "source_suggestion": str,       # Which suggestion created this
    "geometry": {
      "coordinates": list,          # [[x1, y1], [x2, y2], ...]
      "dimensions": dict,           # {"width": float, "height": float}
      "rotation": float,            # Rotation angle in degrees
      "metadata": dict              # Custom properties
    },
    "properties": {
      "name": str,                  # Shape name
      "material": str,              # Material type
      "color": str,                 # Hex color code
      "layer": str                  # GH layer assignment
    },
    "constraints": list,            # Applied constraints
    "metadata": {
      "confidence": float,
      "created_at": str,
      "version": int
    }
  }
  ```
- **Populated By**: Shape Creation node
- **Used By**: Constraint Check, Evaluation, Visualization nodes

#### `shape_creation_report: str`
- **Purpose**: Summary of shape creation process
- **Contents**: 
  - Number of shapes created
  - Creation method/algorithm used
  - Any warnings or issues
  - Processing time
- **Example**: `"Successfully created 3 shapes from suggestion analysis in 0.45s"`
- **Updated By**: Shape Creation node
- **Used By**: Final report generation

#### `passes_constraints: bool`
- **Purpose**: Overall constraint validation result
- **Values**: 
  - `True`: All proposed shapes pass constraints
  - `False`: One or more constraint violations detected
- **Set By**: Constraint Check node
- **Used By**: Routing logic, Evaluation node, Optimization node
- **Impact**: Determines flow to Optimization vs Evaluation

#### `constraint_violations: list[str]`
- **Purpose**: Detailed list of constraint failures
- **Structure**: Each entry is a violation description
- **Examples**:
  ```python
  [
    "Island 'prep_counter' overlaps with room boundary",
    "Door swing clearance insufficient (need 0.9m, have 0.7m)",
    "Appliance placement violates ventilation constraint"
  ]
  ```
- **Initial**: Empty list `[]`
- **Set By**: Constraint Check node
- **Used By**: Optimization node (to generate fixes), Feedback loop
- **Empty When**: All constraints satisfied

#### `constraint_report: str`
- **Purpose**: Detailed constraint validation report
- **Contents**:
  - Total constraints checked
  - Constraints passed/failed
  - Severity of violations (minor/major/critical)
  - Recommendations for fixes
- **Example**: `"Validation: 12 constraints checked. 10 passed, 2 failed (minor)"`
- **Set By**: Constraint Check node
- **Used By**: Final report, User feedback

#### `performance_metrics: dict[str, float]`
- **Purpose**: Quantified design performance scores
- **Standard Metrics**:
  ```python
  {
    "area_efficiency": 0.82,              # Usable area / total area
    "constraint_satisfaction": 1.0,       # Constraints met (0-1)
    "geometry_quality": 0.87,             # Shape quality score
    "optimization_potential": 0.65,       # Room for improvement
    "workflow_efficiency": 0.79,          # User movement optimization
    "accessibility_score": 0.90,          # ADA compliance
    "cost_efficiency": 0.75                # Material/labor efficiency
  }
  ```
- **Range**: Each metric is normalized to [0.0, 1.0]
- **Calculation**: Algorithm-specific (see Evaluation node)
- **Used By**: Decision making, Optimization logic
- **Aggregation**: Average score used for optimization_needed flag

#### `evaluation_report: str`
- **Purpose**: Detailed performance evaluation summary
- **Contents**:
  - Individual metric scores
  - Average performance
  - Performance trends (improving/stable/declining)
  - Identified weak areas
  - Recommendations
- **Example**: `"Performance evaluation: Avg score 82%. Strong areas: constraint satisfaction (100%), weak area: cost efficiency (75%)"`
- **Set By**: Evaluation node
- **Used By**: Final report, Decision making

#### `optimization_needed: bool`
- **Purpose**: Flag indicating if optimization is beneficial
- **Logic**: `True` if average performance < 85%
- **Set By**: Evaluation node
- **Used By**: Routing decision (`_route_after_evaluation`)
- **Impact**: Determines if Optimization node runs

#### `optimization_suggestions: list[dict[str, Any]]`
- **Purpose**: Actionable improvement recommendations
- **Structure**: Each suggestion contains:
  ```python
  {
    "id": str,                      # Suggestion ID
    "type": str,                    # "geometry_refinement", "reposition", etc.
    "description": str,             # What to improve
    "priority": str,                # "high", "medium", "low"
    "expected_impact": {
      "metric": str,                # Which metric improves
      "before": float,              # Current value
      "after": float,               # Projected value
      "improvement_percent": float   # Percentage improvement
    },
    "implementation": str,          # How to implement
    "cost_estimate": str,           # Resource cost
    "complexity": str               # "simple", "moderate", "complex"
  }
  ```
- **Populated By**: Optimization node
- **Example**:
  ```python
  {
    "id": "opt_001",
    "type": "geometry_refinement",
    "description": "Adjust island dimensions for better workflow",
    "priority": "high",
    "expected_impact": {
      "metric": "workflow_efficiency",
      "before": 0.72,
      "after": 0.88,
      "improvement_percent": 22.2
    }
  }
  ```
- **Used By**: Visualization, Final report
- **Empty When**: Optimization not needed

#### `optimization_report: str`
- **Purpose**: Summary of optimization analysis
- **Contents**:
  - Number of suggestions generated
  - Total improvement potential
  - Top 3 recommended optimizations
  - Implementation complexity estimate
- **Example**: `"Generated 4 optimization suggestions. Total potential improvement: 18%. Top priority: geometry refinement (high impact, simple)"`
- **Set By**: Optimization node
- **Used By**: Final report

#### `user_decision: Literal["accept", "modify", "reject", "optimize"]`
- **Purpose**: Track user's decision on proposed design
- **Values**:
  - `"accept"`: User approves design as-is
  - `"modify"`: User wants modifications (returns to Suggestion Layer)
  - `"reject"`: User rejects design completely
  - `"optimize"`: User requests optimization cycle
- **Default**: `"accept"`
- **Set By**: Feedback loop (typically in interactive mode)
- **Used By**: Routing logic (`_route_after_decision`)
- **Impact**: Determines workflow continuation

#### `decision_reason: str`
- **Purpose**: Justification for user decision
- **Contents**: User's explanation or feedback
- **Example**: `"Accept - workflow efficiency improved significantly"`
- **Set By**: User feedback collection
- **Used By**: Logging, Future analysis

### PHASE 3: DECISION OUTPUT & FEEDBACK

#### `human_feedback: str`
- **Purpose**: Free-form user feedback on design
- **Contents**: Any comments, concerns, or suggestions from user
- **Empty When**: No feedback provided
- **Set By**: User interaction (feedback loop)
- **Used By**: Modification logic, Future design considerations
- **Example**: `"Kitchen island is too large, workspace feels cramped"`

#### `feedback_received: bool`
- **Purpose**: Flag indicating user provided feedback
- **Values**: `True` if `human_feedback` is non-empty
- **Set By**: Feedback collection node
- **Used By**: Conditional logic in optimization

#### `reasoning: str`
- **Purpose**: LLM-generated justification for design choices
- **Contents**:
  - Why this design was selected
  - How it addresses user requirements
  - Constraint satisfaction explanation
  - Performance justification
- **Example**: 
  ```
  "This design was selected because it maximizes workflow efficiency 
   (78%) while maintaining all safety constraints. The island placement 
   provides optimal prep-to-cooking distance of 1.8m."
  ```
- **Generated By**: Reasoning node
- **Used By**: Final output, Visualization

#### `why_reasoning: str`
- **Purpose**: Narrative explanation of design rationale
- **Contents**:
  - User requirements addressed
  - Design principles applied
  - Trade-offs made
  - Long-term design benefits
- **Length**: Multi-paragraph narrative
- **Generated By**: Reasoning node
- **Used By**: User communication

#### `visualization_data: dict[str, Any]`
- **Purpose**: Data formatted for Grasshopper visualization
- **Structure**:
  ```python
  {
    "proposed_shapes": list,        # Shapes in GH format
    "metrics": dict,                # Performance chart data
    "constraints_visualization": {
      "passed": list,               # Satisfied constraints
      "failed": list,               # Violated constraints
      "visualization_objects": list # GH geometry for display
    },
    "recommendations": list,        # Visual indicators for improvements
    "viewport": {
      "center": [x, y],
      "zoom": float,
      "layers": dict                # Layer assignments
    }
  }
  ```
- **Generated By**: Visualization node
- **Used By**: Grasshopper display, Report generation

#### `final_shapes: list[dict[str, Any]]`
- **Purpose**: Final approved shapes for implementation
- **Source**: Copied from `proposed_shapes` after acceptance
- **Structure**: Same as `proposed_shapes`
- **Set By**: Output node
- **Used By**: Grasshopper export, GH file generation

#### `final_response: str`
- **Purpose**: Complete response message to user
- **Contents**:
  - Summary of design generated
  - Key improvements achieved
  - Next steps or recommendations
  - Links to visualization/files
  - Timeout or tool-execution notes when MCP/Grasshopper responds too slowly
- **Example**:
  ```
  "Design iteration 5 complete. Generated 3 shapes achieving:
   - 82% area efficiency (target: 80%)
   - 100% constraint satisfaction
   - 18% workflow improvement over previous
   
   Recommendations: Consider material upgrades for 4% cost savings.
   Visualization ready in Grasshopper."
  ```
- **Generated By**: Output node
- **Returned To**: User via `run_agent()` return value

#### `final_scene_state: dict[str, Any]`
- **Purpose**: Complete state snapshot for caching/persistence
- **Structure**:
  ```python
  {
    "shapes": list,                 # Final shapes
    "metrics": dict,                # Performance metrics
    "constraints_satisfied": bool,  # Overall validation
    "timestamp": str,               # Generation time
    "iteration": int,               # Iteration count
    "user_prompt": str,             # Original user request
    "layout_snapshot": dict,        # Layout state at time
    "performance_history": list,    # Metric progression
    "decision_history": list        # All user decisions made
  }
  ```
- **Set By**: Output node
- **Purpose**: Enable resuming/comparing designs later
- **Used By**: Caching system, Historical analysis

### RUNTIME METADATA

#### `iteration: int`
- **Purpose**: Current workflow iteration count
- **Initial Value**: `0`
- **Increments**: +1 at start of each major phase
- **Max Value**: `max_iterations` (safety limit)
- **Used By**: Logging, Debugging, Loop prevention
- **Visible In**: Final reports

#### `max_iterations: int`
- **Purpose**: Safety limit to prevent infinite loops
- **Default Value**: Loaded from environment/config
- **Typical Value**: `3` to `5`
- **Set By**: Bootstrap context
- **Used By**: Routing logic for loop-back prevention

#### `request_timeout_seconds: float`
- **Purpose**: Maximum time allowed for a single MCP request before the workflow treats it as a timeout
- **Default Value**: Loaded from environment/config, currently higher than the earlier 30 second default
- **Set By**: Runtime configuration
- **Used By**: MCP client creation and workflow timeout handling

#### `loop_count: int`
- **Purpose**: Counter for modification loops
- **Increments**: +1 when `user_decision == "modify"`
- **Reset**: When moving to Reasoning phase
- **Purpose**: Prevent infinite modify cycles
- **Used By**: Conditional routing

#### `tool_catalog: str`
- **Purpose**: Formatted list of available MCP tools
- **Format**: Plain text with tool descriptions
- **Example**:
  ```
  - create_shape: Creates geometric shapes | inputSchema={...}
  - validate_constraints: Checks constraints | inputSchema={...}
  - calculate_metrics: Computes performance | inputSchema={...}
  ```
- **Set By**: `_format_tool_catalog()` helper
- **Used By**: LLM (injected into prompts)
- **Read-Only**: Yes

#### `pending_tool_calls: list[dict[str, Any]] | None`
- **Purpose**: Queue of MCP tool calls to execute
- **Structure**: List of {"tool": str, "params": dict} objects
- **Populated By**: Reasoning node (LLM output)
- **Used By**: Tool execution node
- **Cleared By**: After tool execution

#### `last_tool_result: str | None`
- **Purpose**: Stores the most recent raw tool output for final response selection
- **Populated By**: Tool execution node
- **Used By**: Output node and workflow return logic

#### `tool_results: list[str]`
- **Purpose**: Collect all tool outputs generated during the run
- **Populated By**: Tool execution node
- **Used By**: Final response resolution

#### `pending_action: str`
- **Purpose**: Current workflow action selected by the reasoning node
- **Common Values**: `suggest`, `evaluate`, `optimize`, `explain`, `visualize`, `ask_user`, `tool`, `final`
- **Used By**: Routing functions

#### `error_state: str | None`
- **Purpose**: Flag and description of any errors
- **Value**: `None` if no error, error message if error occurred
- **Set By**: Any node encountering exception
- **Used By**: Error handling, Logging, Graceful degradation
- **Examples**:
  - `"MCP connection failed"`
  - `"Invalid shape geometry: negative dimensions"`
  - `"Constraint validation timeout"`

## State Transitions by Node

### Input Setup Node
**Reads**: `user_prompt`, `max_iterations`
**Writes**: `messages`, `iteration`, `loop_count`, `suggestions`, `current_suggestion_index`

### Suggestion Layer Node
**Reads**: `layout_data`, `messages`, `tool_catalog`
**Writes**: `suggestions`, `current_suggestion_index`

### Shape Creation Node
**Reads**: `suggestions`, `current_suggestion_index`
**Writes**: `proposed_shapes`, `shape_creation_report`

### Constraint Check Node
**Reads**: `proposed_shapes`, `layout_data`
**Writes**: `passes_constraints`, `constraint_violations`, `constraint_report`

### Evaluation Node
**Reads**: `proposed_shapes`, `passes_constraints`
**Writes**: `performance_metrics`, `evaluation_report`, `optimization_needed`

### Optimization Node
**Reads**: `performance_metrics`, `proposed_shapes`
**Writes**: `optimization_suggestions`, `optimization_report`

### Reasoning Node
**Reads**: All analysis results
**Writes**: `reasoning`, `why_reasoning`, `pending_action`, `pending_tool_calls`, `generated_options`

### Visualization Node
**Reads**: `proposed_shapes`, `performance_metrics`, `constraint_violations`
**Writes**: `visualization_data`

### Output Node
**Reads**: All state variables
**Writes**: `final_shapes`, `final_response`, `final_scene_state`, `last_tool_result`

## State Validation Rules

1. **Consistency**: `len(proposed_shapes) > 0` when not in initial stage
2. **Metrics**: All values in `performance_metrics` must be in [0.0, 1.0]
3. **Indices**: `current_suggestion_index` must be valid for `suggestions` length
4. **Constraints**: `passes_constraints = (len(constraint_violations) == 0)`
5. **Routing**: `user_decision` must be one of allowed literal values
6. **Iteration**: `iteration <= max_iterations` (safety check)
7. **Timeout**: MCP tool timeouts are surfaced as a final response instead of crashing the notebook

## Extension: Adding Custom State Variables

To add new state variables:

1. Add to `AgentState` TypedDict:
   ```python
   custom_metric: float = 0.0
   ```

2. Initialize in `_build_initial_state()`:
   ```python
   "custom_metric": 0.0,
   ```

3. Modify in relevant node:
   ```python
   state["custom_metric"] = calculate_custom_metric()
   ```

4. Document in this reference
